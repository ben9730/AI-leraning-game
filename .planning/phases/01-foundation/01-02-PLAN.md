   - Import MMKV from react-native-mmkv
   - Create and export `const storage = new MMKV()`
   - Create and export `class MMKVAdapter implements PersistenceAdapter` with methods:
     - `get(key)` returns `storage.getString(key) ?? null`
     - `set(key, value)` calls `storage.set(key, value)`
     - `remove(key)` calls `storage.delete(key)`
     - `clear()` calls `storage.clearAll()`
   - Also export a `zustandMMKVStorage` object conforming to Zustand's `StateStorage` interface:
     ```typescript
     export const zustandMMKVStorage: StateStorage = {
       setItem: (name, value) => storage.set(name, value),
       getItem: (name) => storage.getString(name) ?? null,
       removeItem: (name) => storage.delete(name),
     }
     ```

1. Create `src/store/types.ts`:
   ```typescript
   export interface XPTransaction {
     amount: number
     source: 'lesson_complete' | 'exercise_pass' | 'streak_bonus' | 'perfection_bonus'
     timestamp: number // Unix ms
   }

   export interface UserProgress {
     // State
     userId: string
     xpTotal: number
     xpHistory: XPTransaction[]
     streakCount: number
     lastActivityDate: string  // ISO YYYY-MM-DD, local timezone
     completedLessons: string[]
     unlockedLessons: string[]
     language: 'en' | 'he'
     dailyGoal: 'casual' | 'regular' | 'serious' | null
     streakFreezes: number

     // Actions
     addXP: (amount: number, source: XPTransaction['source']) => void
     completeLesson: (lessonId: string) => void
     unlockLesson: (lessonId: string) => void
     setLanguage: (lang: 'en' | 'he') => void
     setDailyGoal: (goal: UserProgress['dailyGoal']) => void
     updateStreak: () => void
   }

   // NEVER store currentLevel — always derive from xpTotal
   export const getLevel = (xp: number): number => {
     const thresholds = [0, 100, 250, 500, 1000, 2000, 3500, 5000]
     return thresholds.filter(t => xp >= t).length
   }
   ```

2. Create `src/store/useProgressStore.ts`:
   - Import create from zustand, persist + createJSONStorage from zustand/middleware
   - Import zustandMMKVStorage from persistence/MMKVAdapter
   - Import UserProgress, XPTransaction from types
   - Create store with persist middleware:
     - Default state: `userId` = generated UUID (use crypto.randomUUID() or a simple generator), `xpTotal: 0`, `streakCount: 0`, `lastActivityDate: ''`, `completedLessons: []`, `unlockedLessons: ['lesson-01-what-is-prompting']`, `language: 'en'`, `dailyGoal: null`, `xpHistory: []`, `streakFreezes: 0`
     - `addXP(amount, source)`: Push to xpHistory, increment xpTotal
     - `completeLesson(lessonId)`: Add to completedLessons if not already present
     - `unlockLesson(lessonId)`: Add to unlockedLessons if not already present
     - `setLanguage(lang)`: Update language field
     - `setDailyGoal(goal)`: Update dailyGoal field
     - `updateStreak()`: Compare lastActivityDate to today's ISO date. If different day, increment streakCount and update lastActivityDate. If same day, no-op. (Full midnight reset logic comes in Phase 3.)
   - Persist config: `name: 'user-progress'`, `storage: createJSONStorage(() => zustandMMKVStorage)`
   - Export a `useHasHydrated` selector or an `onRehydrateStorage` callback that sets a `_hasHydrated` flag, so the root layout can gate rendering until store is ready.
