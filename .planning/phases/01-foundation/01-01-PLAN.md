2. Install all Phase 1 dependencies in one batch:
   ```

   npm install i18next react-i18next
   ```

3. Install test framework:
   ```
   npm install --save-dev jest @types/jest ts-jest @testing-library/react-native @testing-library/jest-native

5. Configure tab files in `app/(tabs)/`:
   - `_layout.tsx`: Three Tabs.Screen entries — `name="index"` (title: Home), `name="skill-tree"` (title: Skill Tree), `name="profile"` (title: Profile). Use `useTranslation()` hook to get `t` function and pass `t('tabs.home')`, `t('tabs.skillTree')`, `t('tabs.profile')` as tab titles so they are translatable.
   - `index.tsx`: Minimal Home screen with a View + Text showing `t('home.title')`. Use `paddingStart: 16` (NEVER paddingLeft).
   - `skill-tree.tsx`: Minimal Skill Tree screen with a View + Text showing `t('skillTree.title')`. Use `paddingStart: 16`.
   - `profile.tsx`: Minimal Profile screen with a View + Text showing `t('profile.title')`. Use `paddingStart: 16`.

6. In `app/_layout.tsx` (root layout): Import `src/i18n/index.ts` as a side-effect import (`import '@/src/i18n'`) to ensure i18next initializes before any component renders. Wrap children in a basic provider structure. Configure the path alias `@/` to point to project root in `tsconfig.json` if not already set by the template.

CRITICAL RTL RULE: Every style property in every file must use `Start`/`End` variants:
- `paddingStart` / `paddingEnd` (never paddingLeft/Right)
- `marginStart` / `marginEnd` (never marginLeft/Right)
- `textAlign: 'left'` is OK for LTR-specific content, but prefer omitting it (default follows locale)
- `flexDirection: 'row'` auto-flips in RTL — do NOT manually reverse it

     "language": { "switchToHebrew": "עברית", "switchToEnglish": "English" }
   }
   ```

3. Create `src/i18n/index.ts`:
   - Import i18next, initReactI18next, Localization, I18nManager, Updates
   - Import en and he JSON files
   - Initialize i18next with `resources: { en: { common: enCommon }, he: { common: heCommon } }`, `fallbackLng: 'en'`, `ns: ['common']`, `defaultNS: 'common'`, `interpolation: { escapeValue: false }`
   - For initial language: Check if a stored language preference exists (read from MMKV once persistence is wired in plan 01-02; for now use `Localization.getLocales()[0]?.languageCode === 'he' ? 'he' : 'en'`)
   - Export `setLanguage(lang: 'en' | 'he')` function:
     a. Call `await i18n.changeLanguage(lang)`
     b. Derive RTL: `const shouldBeRTL = lang === 'he'`
     c. IMPORTANT: Do NOT check `I18nManager.isRTL` — it has a known bug (expo/expo#34225). Instead, compare against the stored language preference.
     d. Call `I18nManager.allowRTL(shouldBeRTL)` and `I18nManager.forceRTL(shouldBeRTL)`
     e. Call `await Updates.reloadAsync()` to apply RTL change (full reload required)
   - Export `isRTL()` helper that returns `i18n.language === 'he'` (never reads I18nManager.isRTL)
   - Export the i18n instance as default

4. Add a language toggle button to the Profile tab screen (`app/(tabs)/profile.tsx`):
   - A Pressable that calls `setLanguage(currentLang === 'en' ? 'he' : 'en')`
   - Label: `t('language.switchToHebrew')` or `t('language.switchToEnglish')` based on current language
   - Style with `paddingStart`, `marginStart` (RTL-safe)

5. Create `jest.config.js` at project root with ts-jest preset and moduleNameMapper for `@/` alias.
