<objective>
Configure i18next with EN+HE translations from shared/, implement a useLanguage hook that switches language + RTL direction, and ensure all Tailwind styling uses CSS logical properties.

Purpose: i18n and RTL are horizontal concerns consumed by every future UI component. After this plan, any component can call `useTranslation()` for localized text and `useLanguage()` to switch language with instant RTL flip. The CSS logical property convention established here prevents physical left/right from ever entering the codebase.

Output: Working i18next with both languages, useLanguage hook with RTL side-effect, Tailwind logical property convention, and a demo language toggle in the app.
</objective>

i18next.use(initReactI18next).init({

Tailwind v4 logical property utilities (no config needed -- built-in):
- ps-* (padding-inline-start) replaces pl-*
- pe-* (padding-inline-end) replaces pr-*
- ms-* (margin-inline-start) replaces ml-*
- me-* (margin-inline-end) replaces mr-*
- start-* replaces left-*, end-* replaces right-*
- text-start replaces text-left, text-end replaces text-right
