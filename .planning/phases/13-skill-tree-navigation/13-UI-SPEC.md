## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding |
| sm | 8px | Compact element spacing, gap between node connector dots |
| md | 16px | Default card padding (`p-4`), horizontal page gutters |
| lg | 24px | Section padding (`p-6`), vertical rhythm between chapters |
| xl | 32px | Major layout gaps between tree sections |
| 2xl | 48px | Above-the-fold breathing room on profile/onboarding screens |
| 3xl | 64px | Page-level top/bottom padding on desktop breakpoints |

Exceptions:
- Tab bar height: 56px (`h-14`) — matches GameHeader height for visual balance
- Skill tree node touch target: 48px minimum diameter — thumb-friendly on mobile
- Tab icon + label touch area: 44px height minimum — accessibility tap target requirement

Source: `GameHeader.tsx` uses `h-14` (56px); `HomePage.tsx` uses `p-4`/`p-6` as the established rhythm.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label | 12px (`text-xs`) | 400 regular | 1.4 |
| Heading | 20px (`text-xl`) | 700 bold | 1.2 |
| Display | 30px (`text-3xl`) | 700 bold | 1.1 |

Notes:
- Body (14px/400) is used for lesson descriptions, badge descriptions, profile stats body
- Label (12px/400) is used for tab labels, lesson state badges ("Completed", "Locked"), chapter progress counts
- Heading (20px/700) is used for chapter headers in skill tree, section headers in profile
- Display (30px/700) is used for page titles (PromptPlay wordmark, onboarding title, profile page title)

Weights used: 400 regular (`font-normal`) and 700 bold (`font-bold`) — exactly 2 weights.

Source: `HomePage.tsx` — `text-sm text-gray-500`, `text-xl font-semibold`, `text-3xl font-bold`; `LevelUpModal.tsx` — `text-2xl font-bold`. Previous `font-semibold` (600) and `font-medium` (500) usages consolidated to `font-bold` (700) and `font-normal` (400) respectively.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` white / `indigo-50` to `purple-50` gradient | Page backgrounds, card surfaces, main content area |
| Secondary (30%) | `gray-50` (`#f9fafb`) + `gray-200` borders | Tab bar surface, locked node backgrounds, secondary cards, chapter header rows |
| Accent (10%) | `indigo-600` (`#4f46e5`) | See reserved list below |
| Streak/Energy | `orange-600` (`#ea580c`) | Streak flame counter in GameHeader only |
| Success | `green-500` (`#22c55e`) | Completed lesson node fill, completed state indicators |
| Locked/Disabled | `gray-300` (`#d1d5db`) + `opacity-60` | Locked skill tree nodes, locked lesson row icons |

Accent (`indigo-600`) reserved for:
1. Primary CTA buttons ("Start Learning", "Continue", goal selection confirm)
2. Unlocked skill tree node borders and pulse animation ring
3. XP counter text in GameHeader
4. Active tab indicator underline/dot in TabBar
5. Onboarding goal card selected state border
6. Tip/callout box background tint (`indigo-50`) and border (`indigo-200`)

Accent is NOT used for: locked nodes, body text, secondary buttons, background fills, tab icons in inactive state.

Source: `GameHeader.tsx` — `text-indigo-600`, `text-orange-600`; `HomePage.tsx` — `border-indigo-200`, `bg-indigo-100`, `border-green-200`, `bg-green-50`, `bg-gray-50`, `border-gray-200`; `LevelUpModal.tsx` — `bg-indigo-600`, `text-indigo-700`.

## Skill Tree Node States

| State | Visual Treatment | Accessibility |
|-------|-----------------|---------------|
| Locked | 48px circle, `bg-gray-300`, lock emoji icon, `opacity-60`, no hover effect, `cursor-not-allowed` | `aria-label="{lesson title} — locked"`, `aria-disabled="true"` |
| Unlocked | 48px circle, `border-2 border-indigo-600`, `bg-white`, lesson order number, pulsing dot indicator (CSS `animate-pulse`), `hover:border-indigo-700 hover:shadow-md`, `cursor-pointer` | `aria-label="{lesson title} — start lesson"` |
| Completed | 48px circle, `bg-green-500`, white checkmark (✓), no pulse, `hover:bg-green-600 cursor-pointer` | `aria-label="{lesson title} — completed"` |

Node connector: 24px vertical line segment (`border-s-2 border-gray-200`) between adjacent nodes in a chapter. Uses `border-s` (logical, RTL-safe). Green (`border-green-300`) when the node below is completed.

## Interaction Contracts

### Touch / Click Targets

| Element | Minimum touch area |
|---------|--------------------|
| Tab bar item | Full tab width × 56px height |
| Skill tree node | 48px × 48px |
| Goal card | Full card width × min 64px height |
| Badge tap (tooltip) | Entire badge card — 100% × 100% of card |

### Keyboard Accessibility

| Element | Keyboard behavior |
|---------|------------------|
| Tab bar items | `role="tab"`, navigable with arrow keys, Enter/Space activates |
| Skill tree unlocked node | `role="button"`, Enter/Space navigates to lesson |
| Skill tree completed node | `role="button"`, `aria-label="{lesson title} — completed"`, Enter/Space re-navigates to lesson |
| Skill tree locked node | `role="presentation"`, not focusable (`tabIndex={-1}`) |
| Goal cards | `role="radio"` within `role="radiogroup"`, arrow keys move selection |
| Onboarding CTA | Standard `<button>`, `disabled` attribute when no goal selected |

### RTL Compliance

All layout uses CSS logical properties (Tailwind v4):
- `ps-*`/`pe-*` not `pl-*`/`pr-*`
- `ms-*`/`me-*` not `ml-*`/`mr-*`
- `text-start` not `text-left`
- `inset-x-0` for TabBar fixed positioning (already logical)
- Node connector line: `border-s-2` (logical) — flips correctly in RTL

### Animation

| Interaction | Animation |
|-------------|-----------|
| Unlocked node | `animate-pulse` on outer ring (Tailwind built-in) |
| Goal card selection | `transition-colors duration-150` on border/background |
| Tab switch | `transition-colors duration-150` on indicator |
| Onboarding page mount | No animation — keep first-load fast for 60-second target |

## Route Structure

| Route | Component | Notes |
|-------|-----------|-------|
| `/` | `HomePage` | Unchanged |
| `/tree` | `SkillTreePage` | New |
| `/profile` | `ProfilePage` | New |
| `/lesson/:id` | `LessonPage` | Unchanged |
| `/onboarding` | `OnboardingPage` | New — redirects to `/` after completion |

RootLayout wraps all routes except `/onboarding` with TabBar. The `/onboarding` route renders without GameHeader and TabBar (full-screen, distraction-free).

TabBar is added to RootLayout below `<main>` and above the fixed-bottom bar. The main content area needs `pb-14` padding-bottom to clear the TabBar on mobile.
