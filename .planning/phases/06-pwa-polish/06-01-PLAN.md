2. Create `PromptPlay/web/service-worker.js` — the hand-authored SW template for InjectManifest:
   - Import from workbox-precaching: `precacheAndRoute(self.__WB_MANIFEST)`
   - Import from workbox-routing: `registerRoute`
   - Import from workbox-strategies: `CacheFirst`, `StaleWhileRevalidate`
   - Import from workbox-expiration: `ExpirationPlugin`
   - Import from workbox-cacheable-response: `CacheableResponsePlugin`
   - Route 1: App shell (script + style destinations) -> CacheFirst, cacheName 'app-shell', maxEntries 60, statuses [200]
   - Route 2: Lesson JSON (url.pathname matches lesson content paths — check how lesson JSON is bundled in the static export; likely under `/assets/` or inline) -> CacheFirst, cacheName 'lesson-content', maxEntries 100, maxAgeSeconds 30 days, statuses [200]
   - Route 3: Images (request.destination === 'image') -> StaleWhileRevalidate, cacheName 'images', maxEntries 60, maxAgeSeconds 7 days, statuses [200]
   - Route 4: Lottie JSON files (url.pathname ending in .json under assets) -> StaleWhileRevalidate, cacheName 'lottie-assets', maxEntries 20, maxAgeSeconds 7 days
   - Add `self.skipWaiting()` in the install event listener
   - Add `clientsClaim()` from workbox-core in the activate handler
   - IMPORTANT: Do NOT set a global maximumFileSizeToCacheInBytes over 5MB — this enforces the iOS 35MB working budget per the research.

3. Create `PromptPlay/webpack.config.js`:
   - Import `createExpoWebpackConfigAsync` from `@expo/webpack-config`
   - Import `InjectManifest` from `workbox-webpack-plugin`
   - Export async function that creates the base Expo webpack config, then pushes InjectManifest plugin:
     - `swSrc: './web/service-worker.js'`
     - `swDest: 'service-worker.js'`
     - `maximumFileSizeToCacheInBytes: 5 * 1024 * 1024` (5MB per file limit)
   - Return the modified config

   - Create `PromptPlay/workbox-config.js` with: `swSrc: './web/service-worker.js'`, `swDest: './dist/service-worker.js'`, `globDirectory: './dist'`, `globPatterns: ['**/*.{js,css,html,json,png,svg}']`, `maximumFileSizeToCacheInBytes: 5 * 1024 * 1024`

  <action>
1. Create `PromptPlay/src/features/pwa/useServiceWorker.ts`:
   - Export `useServiceWorker()` hook
   - Guard: if `Platform.OS !== 'web'` or `typeof window === 'undefined'`, return early (no-op on native)
   - Guard: if `!('serviceWorker' in navigator)`, return early
   - Use `useEffect` (runs once on mount) to add a `window.load` event listener that:
     - Calls `navigator.serviceWorker.register('/service-worker.js')`
     - On the returned registration, listens for `updatefound` event
     - When a new SW is `installed` and `navigator.serviceWorker.controller` already exists (meaning this is an update, not first install), set an `updateAvailable` state to true
   - Return `{ updateAvailable, applyUpdate }` where `applyUpdate` calls `window.location.reload()`
   - IMPORTANT: Registration MUST be deferred to after `window.load` to avoid blocking first paint (per research Pitfall 4)

2. Wire into `PromptPlay/app/_layout.tsx`:
   - Import `useServiceWorker` from `@/src/features/pwa/useServiceWorker`
   - Call `useServiceWorker()` inside `RootLayoutNav` (NOT inside `RootLayout` — it must be inside the rendered component tree, after hydration gate)
   - For now, just call the hook. The `updateAvailable` state will be consumed by an update banner in a future iteration (not in scope for this plan)

3. Create test file `PromptPlay/src/features/pwa/__tests__/useServiceWorker.test.ts`:
   - Mock `Platform.OS` as 'web'
   - Mock `navigator.serviceWorker` with `register` returning a mock registration object
   - Mock `window.addEventListener` to capture the 'load' callback
   - Test 1: Hook returns `{ updateAvailable: false }` initially
   - Test 2: After firing the 'load' event, `navigator.serviceWorker.register` is called with '/service-worker.js'
   - Test 3: When Platform.OS is not 'web', register is NOT called
