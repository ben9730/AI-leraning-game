1. Create `PromptPlay/src/features/pwa/useInstallPrompt.ts`:
   - Web-only hook that captures the `beforeinstallprompt` event
   - Guard: if `Platform.OS !== 'web'`, return `{ canInstall: false, triggerInstall: async () => {} }`
   - In `useEffect`, add `window.addEventListener('beforeinstallprompt', handler)` where handler calls `e.preventDefault()` and stores the event in state
   - `triggerInstall()` calls `promptEvent.prompt()`, awaits `promptEvent.userChoice`, then clears the event
   - Return `{ canInstall: boolean, triggerInstall: () => Promise<void> }`

2. Create `PromptPlay/src/features/pwa/InstallBanner.tsx`:
   - A dismissible banner component that handles BOTH iOS and Android install flows
   - Detection logic:
     - `isIOSSafari()`: `Platform.OS === 'web'` AND `/iphone|ipad|ipod/i.test(navigator.userAgent)` AND NOT in standalone mode
     - `isStandalone()`: `('standalone' in navigator && (navigator as any).standalone === true)` OR `window.matchMedia('(display-mode: standalone)').matches`
   - Use `useInstallPrompt` hook for Android
   - Render logic:
     - If `isStandalone()` -> render nothing (already installed)
     - If Android and `canInstall` -> show "Install PromptPlay" button that calls `triggerInstall()`
     - If iOS Safari -> show instructional text: "Install PromptPlay: tap Share, then 'Add to Home Screen'"
     - Otherwise -> render nothing
   - Dismissible: Pressable dismiss button, state tracks `dismissed`
   - Persist dismiss in localStorage: `localStorage.setItem('install-banner-dismissed', 'true')` — check on mount so banner does not reappear after dismiss
   - Style with RTL-safe properties: `paddingStart/End`, `marginStart/End` (per CLAUDE.md convention)
   - Use the app's theme colors (theme_color: #4F46E5 indigo)
   - Use `t('key')` for translatable strings — add install banner keys to i18n:
     - `pwa.installButton`: "Install PromptPlay" / Hebrew equivalent
     - `pwa.iosInstallHint`: "Install PromptPlay: tap Share, then 'Add to Home Screen'" / Hebrew equivalent
     - `pwa.dismiss`: "Dismiss" / Hebrew equivalent

3. Wire `InstallBanner` into `PromptPlay/app/(tabs)/_layout.tsx`:
   - Import and render `<InstallBanner />` above the `<Tabs>` component
   - The banner should appear at the top of the tab layout, pushing tabs content down when visible

4. Create test `PromptPlay/src/features/pwa/__tests__/InstallBanner.test.tsx`:
   - Mock Platform.OS, navigator.userAgent, navigator.standalone, window.matchMedia
   - Test 1: On iOS Safari (not standalone) -> renders install hint text
   - Test 2: When already in standalone mode -> renders nothing
   - Test 3: After dismiss button pressed -> renders nothing
   - Test 4: When dismissed and remounted -> still renders nothing (localStorage persisted)
   - Test 5: On non-iOS web with canInstall true -> renders install button
   - These tests go in the existing `pwa` jest project (jsdom environment from 06-01)

5. Add a cache budget validation script to package.json:
   ```json
   "scripts": {
     "check:cache-budget": "expo export --platform web && node -e \"const {execSync} = require('child_process'); const size = parseInt(execSync('du -sb dist/ 2>/dev/null || echo 0').toString().split('\\t')[0]); const mb = size / 1024 / 1024; console.log('Build size: ' + mb.toFixed(1) + 'MB / 35MB budget'); if(mb > 35) { console.error('OVER BUDGET'); process.exit(1); }\""
   }
   ```
   Or a simpler cross-platform version that works on Windows:
   ```json
   "check:cache-budget": "expo export --platform web && node -e \"const fs=require('fs');const path=require('path');function dirSize(d){let s=0;for(const f of fs.readdirSync(d,{withFileTypes:true})){const p=path.join(d,f.name);s+=f.isDirectory()?dirSize(p):fs.statSync(p).size;}return s;}const mb=dirSize('dist')/1024/1024;console.log(mb.toFixed(1)+'MB / 35MB budget');if(mb>35)process.exit(1);\""
   ```
