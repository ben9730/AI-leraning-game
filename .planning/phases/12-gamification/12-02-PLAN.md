    **Modify the `handleContinue` callback** — inside the `if (!completionHandledRef.current)` block, BEFORE calling store actions, snapshot the current badge state, then after store actions, diff badges to detect newly earned (research pattern 4 — badge diffing in callback, NOT during render):
    ```tsx
    if (!completionHandledRef.current) {
      completionHandledRef.current = true

      // Snapshot pre-completion badge state
      const preState = useProgressStore.getState()
      const preBadges = deriveBadges(
        preState.completedLessons, preState.peakStreak,
        preState.xpTotal, preState.streakFreezeUsedEver
      )

      // Fire store actions
      completeLesson(lesson.id)
      const allIds = getAllLessonIds()
      const idx = allIds.indexOf(lesson.id)
      if (idx !== -1 && idx + 1 < allIds.length) {
        unlockLesson(allIds[idx + 1])
      }
      addXP(lesson.xpReward, 'lesson_complete')
      updateStreak()

      // Read post-completion state synchronously
      const postState = useProgressStore.getState()
      const postBadges = deriveBadges(
        postState.completedLessons, postState.peakStreak,
        postState.xpTotal, postState.streakFreezeUsedEver
      )

      // Compute actual XP earned (difference, accounts for streak multiplier if addXP applies it)
      const xpEarned = postState.xpTotal - preState.xpTotal
      setCompletionXP(xpEarned)
      setBadges(postBadges)

      // Detect newly earned badges
      const newBadges = postBadges.filter((b, i) => b.earned && !preBadges[i].earned)
      if (newBadges.length > 0) {
        setNewlyEarnedBadge(newBadges[0])
      }
    }
    ```

    NOTE on XP: Looking at the store's addXP, it adds `amount` directly (no calcXP call — the base amount IS the amount). So `xpEarned` will equal `lesson.xpReward`. Use the diff approach anyway for correctness and future-proofing.
