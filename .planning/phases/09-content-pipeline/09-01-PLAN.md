  <action>
    Create web/src/content/loader.ts (CONT-01):
    - Use `import.meta.glob('@shared/content/lessons/*.json', { eager: true })` to load all lesson JSONs at build time
    - The glob returns Record<string, { default: unknown }> — extract the default export from each module
    - Build a `lessons: Record<string, Lesson>` map by extracting the lesson filename (without path and .json extension) as the key
    - Key extraction: from a glob path like `../../shared/src/content/lessons/lesson-01-what-is-prompting.json`, extract `lesson-01-what-is-prompting` using the last path segment minus `.json`
    - Export `loadLesson(id: string): Lesson` that looks up by ID, throws descriptive Error if not found
    - Export `getAllLessonIds(): string[]` that returns Object.keys(lessons) sorted by the lesson's order field
    - Import the Lesson type from '@shared/content/schema' for type safety (not `any`)
    - DO NOT use dynamic import() — use eager: true so lessons are available synchronously (same behavior as v1)

    Create web/src/content/index.ts (CONT-02):
    - Re-export { loadLesson, getAllLessonIds } from './loader'
    - Re-export { chapters, curriculum } from '@shared/content/curriculum'
    - Re-export { Lesson, Chapter, Exercise, LocalizedString } types from '@shared/content/schema'
    - This barrel file is the single import point for all content in web/
  </action>
