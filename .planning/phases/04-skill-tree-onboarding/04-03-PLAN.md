Supabase RLS SQL from research:
```sql
CREATE TABLE user_progress (
  user_id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  xp_total INTEGER NOT NULL DEFAULT 0,
  streak_count INTEGER NOT NULL DEFAULT 0,
  completed_lessons TEXT[] NOT NULL DEFAULT '{}',
  unlocked_lessons TEXT[] NOT NULL DEFAULT '{}',
  synced_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
ALTER TABLE user_progress ENABLE ROW LEVEL SECURITY;
-- 4 policies: select, insert, update, delete all with user_id = auth.uid()
```

    **2. Create `PromptPlay/src/lib/supabase.ts`:**
    - Supabase client singleton using `createClient` from `@supabase/supabase-js`
    - SecureStore adapter: `{ getItem: SecureStore.getItemAsync, setItem: SecureStore.setItemAsync, removeItem: SecureStore.deleteItemAsync }`
    - Config: `auth: { storage: ExpoSecureStoreAdapter, autoRefreshToken: true, persistSession: true, detectSessionInUrl: false }`
    - Read URL and key from `process.env.EXPO_PUBLIC_SUPABASE_URL` and `process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY`

    **3. Create `PromptPlay/src/features/auth/useAuth.ts`:**
    - Custom hook that manages auth state
    - State: `session: Session | null`, `loading: boolean`, `error: string | null`
    - On mount: `supabase.auth.getSession()` to check existing session
    - Subscribe to `supabase.auth.onAuthStateChange()` for token refresh and sign-out events
    - Export functions:
      - `signUp(email: string, password: string)`: calls `supabase.auth.signUp({ email, password })`. On success, calls `syncProgressToCloud(session.user.id)`. Returns `{ error }`.
      - `signIn(email: string, password: string)`: calls `supabase.auth.signInWithPassword({ email, password })`. On success, calls `restoreProgressFromCloud(session.user.id)` then `syncProgressToCloud(session.user.id)`. Returns `{ error }`.
      - `signOut()`: calls `supabase.auth.signOut()`. Does NOT clear local progress (local-first principle).
    - Clean up subscription on unmount

    **4. Create `PromptPlay/src/features/auth/syncProgress.ts`:**
    - `syncProgressToCloud(userId: string)`: Reads `xpTotal, streakCount, completedLessons, unlockedLessons` from `useProgressStore.getState()`. Upserts to `user_progress` table with `user_id = userId` and `synced_at = new Date().toISOString()`.
    - `restoreProgressFromCloud(userId: string)`: Selects from `user_progress` where `user_id = userId`. If data exists AND local `completedLessons.length === 0` (fresh install), calls `useProgressStore.setState({ xpTotal, streakCount, completedLessons, unlockedLessons })`. After restore, ensures `lesson-01-what-is-prompting` is in `unlockedLessons` (pitfall 5 from research).
    - If local already has progress (same device sign-in), do NOT overwrite — local wins per conflict resolution rule.
    - Both functions are async, return `{ error }` or `{ success }`.

    **5. Create `PromptPlay/src/features/auth/AuthScreen.tsx`:**
    - Full-screen component (not a route — rendered as a modal/overlay from LessonScreen or navigated to)
    - Two modes: sign-up and sign-in, togglable with a text link
    - Fields: email (TextInput, keyboardType: 'email-address', autoCapitalize: 'none'), password (TextInput, secureTextEntry: true)
    - Minimum password length: 6 characters (Supabase default)
    - Submit button calls `useAuth().signUp` or `signIn`
    - Show loading spinner during API call
    - Show error message from Supabase on failure (e.g., "User already registered", "Invalid login credentials")
    - On success: call `onSuccess()` callback prop to dismiss and return to app
    - Props: `{ onSuccess: () => void, onCancel: () => void }`
    - RTL: `paddingStart/End`, `textAlign` from `isRTL()`. All text via `useTranslation()`.
    - Encouraging header: "Create your account" / "Welcome back" (not "Sign up required")
