1. Install dependencies from web/ directory:
   ```bash
   cd web && npm install --save-dev vite-plugin-pwa workbox-window && npm install react-helmet-async
   ```

2. Update `web/vite.config.ts` — add VitePWA import and plugin call after tailwindcss():
   ```typescript
   import { VitePWA } from 'vite-plugin-pwa'
   ```
   Add to plugins array (after tailwindcss()):
   ```typescript
   VitePWA({
     registerType: 'prompt',
     includeAssets: ['icons/pwa-192x192.png', 'icons/pwa-512x512.png', 'og-image.png'],
     manifest: {
       name: 'PromptPlay',
       short_name: 'PromptPlay',
       description: 'Learn AI prompting through gamified practice',
       theme_color: '#4f46e5',
       background_color: '#ffffff',
       display: 'standalone',
       orientation: 'any',
       start_url: '/',
       icons: [
         { src: 'icons/pwa-192x192.png', sizes: '192x192', type: 'image/png' },
         { src: 'icons/pwa-512x512.png', sizes: '512x512', type: 'image/png', purpose: 'any maskable' },
       ],
     },
     workbox: {
       navigateFallback: 'index.html',
       runtimeCaching: [
         {
           urlPattern: /\.(js|css|woff2?)(\?.*)?$/,
           handler: 'StaleWhileRevalidate',
           options: {
             cacheName: 'static-assets',
             expiration: { maxEntries: 100 },
           },
         },
         {
           urlPattern: /\.(png|svg|ico|webp)(\?.*)?$/,
           handler: 'CacheFirst',
           options: {
             cacheName: 'image-assets',
             expiration: { maxEntries: 30, maxAgeSeconds: 30 * 24 * 60 * 60 },
           },
         },
       ],
     },
   })
   ```

3. Update `web/src/vite-env.d.ts` to add PWA client types:
   ```typescript
   /// <reference types="vite/client" />
   /// <reference types="vite-plugin-pwa/client" />
   ```

IMPORTANT: Use `registerType: 'prompt'` (NOT 'autoUpdate') — the user decision requires a custom update toast that asks before reloading. The `navigateFallback: 'index.html'` is critical for offline SPA navigation. Lesson JSON is loaded via import.meta.glob with eager:true so it is bundled into JS chunks — the app shell precache covers offline lesson access automatically; no separate JSON runtimeCaching rule needed (per RESEARCH.md Open Question 1).
