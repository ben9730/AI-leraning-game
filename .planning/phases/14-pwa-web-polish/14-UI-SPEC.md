## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between tight elements |
| sm | 8px | Compact element spacing, gap between badge items |
| — | 12px (gap-3) | Legacy value present in existing components — not newly introduced in phase 14; do not add new usage |
| md | 16px | Default card padding, horizontal page padding (p-4) |
| lg | 24px | Section vertical spacing (space-y-6 = 24px) |
| xl | 32px | Page bottom padding (pb-8), major section breaks |
| 2xl | 48px | Reserved — not used in this phase |
| 3xl | 64px | Reserved — not used in this phase |

Exceptions:
- TabBar/Sidebar height: 56px (h-14) — established in TabBar.tsx, do not change
- GameHeader height: 56px (h-14) — established in GameHeader.tsx, do not change
- Desktop sidebar width: 256px (w-64) on lg+ screens
- Desktop content max-width: 672px (max-w-2xl) centered in remaining space
- Touch targets: minimum 44px height for all interactive banner buttons (install, dismiss)

> Source: codebase scan — existing components use p-4 (16px), gap-3 (12px), space-y-6 (24px). 8-pt scale is already the de facto standard.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (font-normal) | 1.5 |
| Label | 14px (text-sm) | 400 (font-normal) | 1.4 |
| Heading | 20px (text-xl) | 700 (font-bold) | 1.2 |
| Display | 30px (text-3xl) | 700 (font-bold) | 1.1 |

Permitted weights: **400 (font-normal)** and **700 (font-bold)** only. font-semibold (600) is not permitted in phase 14 — use font-bold (700) wherever emphasis is required.

> Source: codebase scan — GameHeader uses text-sm; HomePage uses text-3xl/font-bold for title, text-xl/font-bold for chapter headings; ChapterSection uses text-xl/font-bold; body copy is consistently text-sm at default weight.

New typography in this phase:
- Banner body (offline/install/update): text-sm (14px), weight 400, line-height 1.5 — matches body role
- Sidebar nav labels: text-sm (14px), weight 700 when active, weight 400 when inactive — matches heading weight pattern for active state

## Responsive Layout Contract

> Source: CONTEXT.md decisions — 3-tier breakpoint system, desktop sidebar conversion, mobile-first.

### Breakpoints

| Tier | Breakpoint | Layout |
|------|-----------|--------|
| Mobile | < 768px | Bottom TabBar (h-14), full-width content, single column |
| Tablet | 768px–1023px (md:) | Bottom TabBar retained, content max-w-lg centered with horizontal padding |
| Desktop | 1024px+ (lg:) | Left sidebar nav (w-64), content area max-w-2xl centered, no bottom TabBar |

### Desktop Layout Structure

```
[ Sidebar w-64 | fixed inset-y-0 start-0 ]  [ Main content | ms-64 ]
  - Logo/brand wordmark (top)                  - GameHeader (full width)
  - Nav links (Home, Skill Tree, Profile)       - Page content (max-w-2xl mx-auto)
  - Skill tree mini-map (bottom of sidebar)
```

GameHeader is the primary visual anchor on all viewports; the sidebar is secondary chrome providing navigation context only.

### TabBar Responsive Conversion

- Mobile/tablet: `fixed bottom-0 inset-x-0 h-14` (current behavior — unchanged)
- Desktop (lg+): `hidden lg:hidden` on TabBar component; `Sidebar` component renders at lg+ with `fixed inset-y-0 start-0 w-64`
- Sidebar nav links use same indigo-600 active indicator — left border `border-s-2 border-indigo-600` for active, `border-s-2 border-transparent opacity-60` for inactive
- RTL: sidebar sits on `end-0` when `dir="rtl"` — use CSS logical properties throughout

### Content Width on Desktop

- GameHeader: full width across main area (not full viewport — excludes sidebar)
- Page content: `max-w-2xl mx-auto px-6` within the main area
- Skill tree mini-map in sidebar: `w-full max-h-48 overflow-y-auto` — scaled-down read-only chapter overview

## PWA & Banner Components Contract

### Offline Banner

- Position: fixed top bar, below GameHeader (top: 56px = h-14), full width, z-30
- Height: auto with py-2 px-4 — no fixed height
- Colors: `bg-amber-50 border-b border-amber-200 text-amber-800`
- Copy: i18n key `pwa.offlineBanner` — "You're offline — cached lessons still available" (EN)
- No dismiss button — disappears automatically when network is restored
- Interaction: none (informational only)
- RTL: text flows naturally with logical text-start alignment

### Update Toast

- Position: fixed `top-16 start-1/2 -translate-x-1/2 z-50` — matches BadgeToast positioning pattern
- Colors: `bg-white border border-indigo-200 rounded-lg shadow-lg`
- Copy: i18n key `pwa.updateAvailable` — "New version available" + CTA "Refresh" (EN)
- CTA: `px-3 py-1 bg-indigo-600 text-white text-sm font-bold rounded-md` button
- Dismiss: secondary "Later" link-style button `text-sm text-gray-500`
- Animation: `animate-scale-pulse` (reuse existing keyframe from globals.css)

### iOS Install Nudge

- Position: fixed bottom above TabBar — `fixed bottom-14 inset-x-0 z-30` on mobile; `fixed bottom-0 inset-x-0 z-30` on desktop (no TabBar)
- Colors: `bg-white border-t border-gray-200 shadow-lg`
- Copy: i18n key `pwa.iosInstallHint` — already in common.json: "Install PromptPlay: tap Share, then 'Add to Home Screen'"
- Dismiss CTA: i18n key `pwa.dismiss` — already in common.json: "Dismiss"
- Install CTA: i18n key `pwa.installButton` — already in common.json: "Install PromptPlay"
- iOS nudge shows on first visit, dismissed state persisted in localStorage key `promptplay_install_dismissed`
- Touch target: minimum h-11 (44px) for all banner buttons

### Android Install Prompt

- Triggered by native `beforeinstallprompt` event
- Reuses iOS nudge banner layout with `pwa.installButton` CTA
- Calls `event.prompt()` on CTA click
- Hides after user accepts or dismisses native dialog

## Accessibility Contract

- Offline banner: `role="status" aria-live="polite"` — screen reader announces when network state changes
- Install banner: `role="complementary"` with `aria-label="Install app"` on the nav element
- Update toast: `role="alert" aria-live="assertive"` — announces immediately when new SW is ready
- Desktop sidebar: `<nav aria-label="Main navigation">` — distinct from bottom TabBar `<nav aria-label="Tab bar">`
- All banner close/dismiss buttons: `aria-label="Dismiss"` explicitly set
- Focus management: when install banner appears, no focus trap — banner is non-modal

## Service Worker Caching Contract

> Source: CONTEXT.md — cache-first for lesson JSONs, network-first for app shell.

| Asset Type | Strategy | Rationale |
|-----------|---------|-----------|
| App shell (HTML, JS, CSS bundles) | NetworkFirst, fallback to cache | Freshness needed for updates |
| Lesson JSON files (`/src/content/lessons/*.json`) | CacheFirst, max-age 7 days | Static content, safe to cache aggressively |
| PWA icons + OG image | CacheFirst, max-age 30 days | Never changes between versions |
| Vite build assets (hashed filenames) | StaleWhileRevalidate | Hash changes on deploy guarantee freshness |

> Note: Caching rules are a CONTEXT.md decision (Claude's Discretion). Values above are defaults based on CONTEXT.md guidance.
